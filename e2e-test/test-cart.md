电商平台购物车模块测试用例

1. 购物车基本功能测试

1.1 商品添加
TC-CART-001: 正常添加商品到购物车
前置条件：用户已登录，商品库存>0
测试步骤：
在商品详情页点击"加入购物车"按钮
查看页面右上角购物车图标角标
点击购物车图标进入购物车页面
预期结果：
显示"添加成功"提示
购物车角标数字+1
购物车页面正确显示该商品
测试数据：正常商品SKU
优先级：P0
TC-CART-002: 同一商品多次添加
测试步骤：
同一商品连续点击3次"加入购物车"
进入购物车查看
预期结果：商品数量累加为3
优先级：P1
1.2 商品数量修改
TC-CART-003: 增加商品数量
前置条件：购物车中有商品A，数量为1
测试步骤：
点击商品A的"+"按钮
再次点击"+"按钮
预期结果：
商品数量变为2，再变为3
小计金额同步更新
优先级：P1
TC-CART-004: 减少商品数量
前置条件：购物车中有商品B，数量为3
测试步骤：
点击商品B的"-"按钮
数量为1时再次点击"-"
预期结果：
数量从3→2→1
数量为1时点击"-"，商品被删除
优先级：P1
TC-CART-005: 手动输入数量
测试步骤：
在数量输入框输入"5"
点击其他区域或按回车
输入"0"并确认
预期结果：
数量更新为5
输入0时商品被删除
优先级：P2
1.3 库存边界测试
TC-CART-006: 超过库存上限
前置条件：商品C库存为10
测试步骤：
购物车中商品C数量为5
修改数量为15
点击"+"增加到12
预期结果：
输入15时提示"库存不足，最大可购10件"
自动修正为最大库存10
优先级：P1
TC-CART-007: 商品库存变为0
测试步骤：
购物车中有商品D
后台将商品D库存设为0
刷新购物车页面
预期结果：显示"库存不足"或"已售罄"标识
优先级：P2
1.4 商品移除
TC-CART-008: 删除单个商品
测试步骤：
点击商品右侧的删除图标
在弹出的确认框中点击"确定"
预期结果：商品从购物车移除
优先级：P1
TC-CART-009: 清空购物车
测试步骤：
点击"清空购物车"按钮
确认操作
预期结果：所有商品被移除
优先级：P2
2. 购物车状态与交互

2.1 商品状态变化
TC-CART-010: 商品下架处理
测试步骤：
购物车中有商品E
后台下架商品E
刷新购物车
预期结果：
显示"商品已下架"
商品变灰不可勾选
优先级：P2
TC-CART-011: 价格变动同步
测试步骤：
购物车中有商品F，价格100元
后台修改商品F价格为80元
刷新购物车
预期结果：价格自动更新为80元
优先级：P2
2.2 多商品操作
TC-CART-012: 全选/反选功能
测试步骤：
点击顶部"全选"复选框
取消某个商品的勾选
再次点击"全选"
预期结果：
全选时所有商品被选中
取消某个商品后，全选状态取消
再次全选恢复全选状态
优先级：P1
TC-CART-013: 批量删除
前置条件：购物车中有3个商品
测试步骤：
勾选其中2个商品
点击"删除选中"
确认操作
预期结果：选中的2个商品被删除
优先级：P1
3. 金额计算
TC-CART-014: 小计金额计算
前置条件：商品G单价50元，数量2；商品H单价30元，数量3
测试步骤：进入购物车页面
预期结果：
商品G小计：50×2=100元
商品H小计：30×3=90元
优先级：P1
TC-CART-015: 总计金额计算
测试步骤：修改商品数量
预期结果：总计=所有选中商品小计之和
优先级：P0
4. 兼容性测试
TC-CART-016: 多端同步
测试步骤：
网页端添加商品到购物车
使用同一账号登录App端
预期结果：App端购物车显示相同商品
优先级：P2
说明：以上用例仅覆盖购物车模块核心功能，包含正常流程、异常处理和边界情况。
